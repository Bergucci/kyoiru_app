# Kyoiru Source Spec

## 0. Requirements Summary

Kyoiru（今日いる）は、「連絡しなくても、今日いることだけ分かる」を軸にしたモバイルアプリである。  
無料版は、友人・恋人・家族と 1 タップでゆるくつながる軽さを主役とする。  
有料版は、必要な相手にだけ見守り機能を強化して提供する。

本仕様で最重要の要件:
- 無料版の主役は軽さであること
- 有料版の主役は安心強化であること
- 無料ユーザーに監視アプリと思わせないこと
- LINE ログイン / LINE 共有を重視すること
- block 済み相手への接触導線は厳格に閉じること
- 見守りは見守る側の課金だけでなく、見守られる側の明示同意が必要であること
- 生存報告は日本時間の朝 6:00 基準で 1 日 1 回であること
- 仕様未確定事項は勝手に補完せず、open questions として分離すること

## 1. プロダクト概要

### 1.1 アプリ名
- Kyoiru（今日いる）

### 1.2 コンセプト
- 連絡しなくても、今日いることだけ分かる

### 1.3 基本思想
- 無料版:
  - 軽い生存報告とゆるいつながりを主軸にする
- 有料版:
  - 見守りが必要な相手に対し、安心強化として提供する

### 1.4 崩してはいけない方針
- 無料版の主役は軽さ
- 有料版の主役は安心強化
- 無料ユーザーに「監視アプリ」と思わせない
- LINE ログイン / LINE 共有を重視する
- block 済み相手への接触導線は厳格に閉じる

## 2. 課金設計

### 2.1 プラン
- プラン名: 見守りプラン
- 価格: 月額 980 円
- トライアル: 7 日間無料トライアルあり
- 課金形態: 月額課金のみ
- 買い切りなし

### 2.2 課金訴求タイミング
課金訴求は、必要性が発生したタイミングでのみ表示する。
- 見守りモードを有効化しようとした時
- 未反応が続いた時
- 親・高齢者用途で使い始める時

### 2.3 解約・期限切れ・Grace Period
- 解約後も App Store / Store の有効期限満了までは有料機能利用可
- 満了後に有料機能停止
- 見守り設定は削除せず保持
- 再契約時に復元可能
- 位置履歴は再契約時に復元しない
- Billing Grace Period は有効化前提
- 決済失敗でも猶予期間中は見守り機能維持
- 猶予期間終了後に停止

## 4. アカウント仕様

### 4.1 ログイン方法
- Apple ログイン
- LINE ログイン
- Google ログイン
- メールアドレス + パスワード

### 4.2 初期プロフィール設定
初回ログイン後に設定する。
- 表示名
- アイコン
- ユーザーID

### 4.3 ユーザーID仕様
- 全ユーザーで一意
- DB レベルで unique 制約必須
- 表示名とは別管理
- ユーザーが自由に設定可能
- 変更は 30 日に 1 回まで
- アカウント設定に次回変更可能日を表示
- 変更時は重複チェック必須

### 4.4 ID 検索公開設定
- アカウント設定に ON/OFF を設ける
- OFF 時:
  - ID 検索結果に表示されない
  - 既存の友達関係・既存グループ関係には影響しない
  - 招待リンク経由では参加可能

## 5. 友達機能仕様

### 5.1 友達追加方法
- ユーザーID検索
- 友達追加用リンク共有
- LINE で送る

### 5.2 友達追加方式
- 承認制

### 5.3 基本フロー
1. 送信側が ID 検索で相手を見つける
2. 友達申請を送る
3. 受信側が承認 / 拒否する
4. 承認後、友達一覧に追加される

### 5.4 ID 検索結果に表示する情報
- アイコン
- 表示名
- ユーザーID

### 5.5 友達申請制限
- 1 日あたり送信上限: 30 件
- 同一相手に対する承認待ち重複申請: 不可
- 拒否された相手への再申請: 30 日間不可
- 承認前なら送信側は申請取消可能

### 5.6 誤拒否対策
- 拒否後 24 時間以内なら拒否取り消し可能
- 拒否取り消し時、申請状態は承認待ちに戻る
- 送信側からの再送は不可のまま

## 6. block 仕様

### 6.1 基本
- block 機能は必須

### 6.2 block 時の基本挙動
- block した相手からの友達申請を受け付けない
- block した相手は ID 検索で自分を見つけられない
- block された相手からも自分を検索できない
- 既存の友達関係は解除される
- 招待候補 / おすすめ導線にも表示しない

### 6.3 block 時のグループ扱い
- block した相手とは新規グループを作成できない
- block した相手を新規グループに招待できない
- block した相手から新規グループに招待されない
- 既存グループに同席している場合は相互表示を制限する

### 6.4 既存グループで残す情報
block 済み相手が既存グループにいる場合、残すのは以下のみ。
- アイコン
- 表示名
- 参加中であることが分かる最低限の存在表示

### 6.5 既存グループで隠す情報
- 最終反応時刻
- 気分スタンプ
- 詳細画面導線
- 見守り関連情報
- 友達状態
- ユーザーID

### 6.6 block と見守り設定の競合
- block が優先
- 既存の見守り設定は停止状態にする
- 再開には block 解除後、再設定が必要

## 7. グループ仕様

### 7.1 初回利用目的選択
- 友人・恋人
- 家族
- 両方

これは利用制限ではなく、初期おすすめ設定である。  
後から友人グループ・家族グループの両方を自由に作成可能。

### 7.2 グループ作成
作成時に設定する。
- グループ名
- 利用タイプ
  - 友人・恋人向け
  - 家族向け
- 任意のグループアイコン

### 7.3 グループ参加
- グループ招待用リンクで参加する
- 招待コードは採用しない

## 8. 招待仕様

### 8.1 招待リンクの種類
- 友達追加用リンク
- グループ招待用リンク

### 8.2 グループ招待リンク共有
- 招待リンクをコピー
- LINE で送る
- その他で共有

### 8.3 LINE 送信
- 「LINE で送る」押下時は LINE アプリへ遷移し、招待文と招待リンクをセットした状態で共有可能にする

### 8.4 招待リンク制約
- 有効期限: 発行から 30 日
- 使用回数: 1 回のみ
- 使い回し不可

### 8.5 無効条件
以下の場合は参加不可。
- 期限切れ
- 使用済み
- block 関係により参加不可

### 8.6 再発行
- グループ管理者が新しい招待リンクを再発行できる
- 再発行時には旧リンクを即無効化
- 最新リンクのみ有効

## 9. 生存報告仕様

### 9.1 基本ルール
- 生存報告は 1 日 1 回のみ
- 日付切り替えは日本時間 朝 6:00
- 基準は端末ローカル日付ではなく日本共通日付

### 9.2 同日再送時
- 2 回目以降は送信不可
- UI 文言は「今日の生存報告は完了しています」
- ただし、気分スタンプ未設定ならスタンプ設定だけは可能
- 生存報告時刻は最初の 1 回を保持

### 9.3 過去日の扱い
- 過去 7 日分の履歴を確認可能
- 過去日は編集不可
- 各日付は「反応済み / 未反応」で表示
- 気分スタンプがある場合はその日付に紐づけて表示

## 10. 気分スタンプ仕様

### 10.1 基本ルール
- 1 日 1 回のみ
- 変更不可
- 削除可
- アカウント共通

### 10.2 設定条件
- 生存報告後にのみ設定可能
- 生存報告していない日は設定不可

### 10.3 削除後
- 削除後の再設定は不可

### 10.4 想定スタンプ
- 元気
- 眠い
- 忙しい
- しんどい
- ふつう

## 11. 状態管理仕様

### 11.1 内部状態と UI 文言の分離
内部状態名と UI 表示文言は分離して扱う。

### 11.2 生存報告状態
- `checked_in` → 今日反応済み
- `pending` → まだ未反応
- `overdue` → 長時間未反応
- `monitor_alert` → 要確認

### 11.3 状態の意味
- `checked_in`
  - 当日分の生存報告が完了している状態
- `pending`
  - 基準日切り替え後、まだ反応していない状態
- `overdue`
  - 一定時間以上反応がない状態
  - 無料版で見える最も強い未反応状態
- `monitor_alert`
  - 見守りモード対象で、段階通知条件に入った状態
  - 主に有料文脈で扱う

## 12. グループ状態表示ロジック

### 12.1 グループ一覧
各グループカードに表示する。
- グループ名
- グループ種別
- メンバー数
- 反応済み人数 / 未反応人数
- 最も強い状態ラベル 1 つ
- 自分が未報告なら「今日いる」導線

表示例:
- 3 人中 2 人反応済み / 1 人未反応
- 全員反応済み
- 1 人長時間未反応

### 12.2 グループ詳細
各メンバーに表示する。
- アイコン
- 表示名
- 状態ラベル
- 最終反応時刻
- 気分スタンプ

### 12.3 並び順
- 自分
- `monitor_alert` / `overdue`
- `pending`
- `checked_in`

同一状態内は、最終反応時刻が古い順を優先してよい。

## 13. 通知温度感仕様

### 13.1 設定単位
- グループごと

### 13.2 設定値
- ゆるい
- ふつう
- 気にかける

### 13.3 各設定の挙動
- ゆるい
  - プッシュ通知なし
  - アプリを開いた時だけ未反応が分かる
- ふつう
  - 翌日の朝 6:00 を過ぎても未反応なら、軽い通知を 1 回送る
- 気にかける
  - 当日 21:00 時点で未反応なら軽い通知
  - 翌日朝 6:00 を過ぎても未反応なら再通知

### 13.4 深夜通知
- 22:00〜7:00 は原則通知しない
- ただし見守りモードの通知は別扱い

## 14. 見守りモード仕様

### 14.1 基本方針
- 見守りモードは相手単位
- 見守る側の課金で有効
- 見守られる側の明示同意が必要
- 見守られる側は無料会員でもよい

### 14.2 有効化フロー
1. 見守る側が対象ユーザーに対して見守り設定開始
2. 対象ユーザーに同意リクエスト送信
3. 対象ユーザーが同意
4. 同意時点で見守りモード有効化

### 14.3 GPS 共有設定
- 共有しない
- 未反応時のみ参照可能
- 常時共有
- 初期値は「未反応時のみ参照可能」

### 14.3.1 位置履歴の保持
- 位置履歴の保持期間は 30 日のローリング保持とする
- 収集は見守りモード有効、同意完了、GPS 共有設定が `未反応時のみ参照可能` または `常時共有` の場合のみ行う
- block、同意撤回、見守り OFF、Grace Period 終了後の課金失効、退会時は新規収集を即時停止する
- 収集停止後、既存の位置履歴と最終位置は 24 時間以内に削除する
- 再契約時は見守り設定のみ復元可能とし、過去の位置履歴は復元しない

### 14.4 緊急連絡先表示条件
以下をすべて満たした場合のみ表示する。
- 見守りモード ON
- 対象ユーザーが事前に緊急連絡先を設定済み
- 段階通知の最終段階に入った時

### 14.4.1 緊急連絡先の保持項目
- 緊急連絡先は 1 件のみ設定可能
- 保持項目は以下とする
  - 氏名: 必須
  - 電話番号: 必須
  - 続柄: 任意
- 住所、メールアドレス、自由記述メモ、複数電話番号は保持しない
- 緊急連絡先は本人が更新・削除可能
- 緊急連絡先の表示時は閲覧ログを残す

### 14.5 段階的エスカレーション
- 第1段階
  - 当日 21:00 時点で未反応
  - 見守る側に軽い通知
- 第2段階
  - 翌日朝 6:00 時点でも未反応
  - 見守る側に強めの通知
  - 位置確認導線表示
- 第3段階
  - 翌日 12:00 時点でも未反応
  - 緊急連絡先表示
  - 「連絡を取ってみてください」導線表示

### 14.6 複数回チェックイン
- 1 日 1 回 / 2 回 / 3 回から選択
- 時間帯はテンプレで選択する
- テンプレ:
  - 朝
  - 朝 / 夜
  - 朝 / 昼 / 夜

### 14.7 見守りダッシュボード表示要素
- 見守り対象一覧
- 今日の反応状況
- 最終位置
- 最終反応時刻
- 緊急連絡先設定有無
- 複数回チェックイン達成状況
- エスカレーション発生状況

### 19.3 今日の生存報告モーダル

  - block 履歴ログ
  - 見守り同意履歴
  - 緊急連絡先閲覧ログ
  - スパム対策ログ

- 退会時はログイン停止、セッショントークン無効化、push 停止、見守り停止、位置収集停止を即時実行する
