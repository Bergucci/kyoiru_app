# Open Questions

このファイルには、現仕様で未確定の論点のみを記載する。
ここにある項目は、決定されるまで実装で勝手に補完しない。

運用ルール:
- 未確定事項は `source-spec.md` に混ぜない
- 実装担当は、未確定事項を独自判断で埋めない
- 判断が確定したら、このファイルを更新し、必要に応じて `source-spec.md`、`architecture.md`、`implementation-plan.md`、`review.md` を更新する
- 技術判断は ADR に記録するが、ユーザー向け仕様の変更は `source-spec.md` に反映する
